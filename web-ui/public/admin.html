Custom News Sources

ⓘ
What is this? Register external sources that the system polls automatically for new articles.
  • RSS Feed — most news sites provide an RSS URL (e.g. https://example.com/feed). This is the most reliable source type.
  • Single URL — fetches one specific article page. Use for one-off items.
  • Use Pause to stop fetching without deleting the source or its articles.
  • Use Fetch Now to pull new items immediately instead of waiting for the schedule.

Manual News Feed

ⓘ
What is this? A manual ingestion lane for content from any source that can't be auto-fetched.
  • Paste forwarded message text directly into the text box, or upload a screenshot / PDF.
  • Set a Source Label to group these articles together in the sentiment dashboard (e.g. "EGX News — Manual").
  • Arabic and English are both supported — language is detected automatically.
  • Submitted content goes through the same sentiment pipeline as auto-fetched articles.
  • You can submit text and a file at the same time — both will be processed.

API Reference

All endpoints return JSON. Auth endpoints require a valid session cookie. Admin endpoints require Authorization: Bearer <token>.

# ── Predictions & Consensus ── GET /api/consensus → array of all stocks with signals GET /api/stocks/:symbol/brief → {brief} Gemini 3-sentence AI narrative GET /api/signal-accuracy?horizon=10 → per-stock accuracy at D+5/D+10/D+20 # ── Prices ── GET /api/prices → latest close price for all symbols GET /api/prices/:symbol → price history for one symbol GET /api/sentiment → latest sentiment per stock GET /api/stats → {total_stocks, latestDate, …} # ── Portfolio & Trades ── GET /api/trades/portfolio → {open_positions, closed_positions, stats, totals, sector_breakdown} POST /api/trades/positions → add position {symbol, entry_price, quantity, entry_date} GET /api/trades/alerts → list alerts (auto-triggers active ones) POST /api/trades/alerts → create alert {symbol, condition, target_price} DELETE /api/trades/alerts/:id → delete alert # ── FX & Rates ── GET /api/fx-rates → {USD_EGP, USD_SAR, SAR_EGP, XAU_USD, …} GET /api/fx-rates/history?days=90 → array of daily rate rows (oldest first) # ── Forecast Portfolios ── GET /api/portfolio-forecasts → list user portfolios POST /api/portfolio-forecasts → create portfolio + run forecast GET /api/portfolio-forecasts/:id/results → per-stock forecast results GET /api/portfolio-forecasts/:id/chart → forecast vs actuals time series # ── ETF ── GET /api/etf/instruments → all ETF instruments GET /api/etf/prices/:symbol → price history GET /api/etf/nav/:symbol → NAV history GET /api/etf/holdings/:symbol → holdings snapshot GET /api/etf/country-exposure → country weights # ── RAG / Chat ── POST /api/rag/chat → {message} → {reply} market assistant POST /api/rag/macro → {} → {answer, sources[]} grounded macro brief POST /api/rag/ask → {question} → {answer} report RAG (admin) # ── Auth ── POST /api/auth/signup → {email, password} → sets cookie POST /api/auth/login → {email, password} → sets cookie POST /api/auth/logout → clears cookie GET /api/auth/me → {id, email} if logged in
Auth note: Portfolio, alerts, and forecast endpoints require a valid session cookie (set by /api/auth/login). Pass credentials: 'include' on all fetch calls from the frontend.
